csv をマージする時の注意点

1
Rn の pdf には、Rn 年度の予算案と、R(n-1)年度の予算（確定済み）が記載されています. それは json や csv から確認できます。
つまり Rn と R(n-1) のcsv を統合する時 予算は Rn案, R(n-1)確定済, R(n-1)案, R(n-2)確定済 が得られるということです。
R(n-1) に重複が起きますが、これは、両方別のデータとして保存しましょう. Rn(案) と Rn(確定) として、csv には入力されるべき。　json -> csv の converter のレイヤーでこれは解決されるべきです。

2
年度によって 予算項目の増減があります。pdf には予算項目に番号が振ってあるのに、json で入力しないようにしたのは、番号は当年度の運用のためのものであり、年度を跨いだ分析には障害になると判断したためです。
予算項目が増えれば、増えた予算案の年度で定義されている位置で、merge 済 csv にカラムを追加しましょう.
つまり、年が経つごとにカラムは増えていきます.
記載のなかった年度では、その項目は 0 として入力しましょう
この辺りは、csv -> merged_csv のレイヤーで解決されるべきです

## マージ実行時に発見した課題

### 3. R8のCSVヘッダーが誤っていた
R8/8年度予算.csv のヘッダーが `R6,R7` となっていたが、実際のデータは `R7(確定),R8(案)` である。
→ `R7,R8` に修正済み

### 4. 説明列の表記ゆれ
年度によって同じ項目でも表記が異なる:
- R6: `均等割（調定見込額 402,000×98.9％）` （カンマあり、×記号）
- R7: `均等割（調定見込額 408000*98.9%）` （カンマなし、*記号）

これにより、同じ項目が別の行として扱われ、データが分散している。
→ 将来的には説明列の正規化が必要

### 5. 階層データの粒度の違い
年度によって記載の詳細度が異なる:
- ある年度では「現年課税分」までの記載
- 別の年度では「現年課税分」の内訳（均等割、所得割）まで記載

→ 分析時に階層レベルを意識する必要がある

### 6. 親行にデータがない場合がある
R7/budget.csvでは、市税などの款レベルの行にR6の値が入っていない（子項目にのみ値がある）。
→ 集計が必要な場合は子項目の合計を計算する処理が必要

## 実装した解決策

### merge_csv.py の機能

1. **項目名と補足の分離**
   - 括弧内を補足として分離
   - 例: `均等割（調定見込額 402,000×98.9％）` → 項目名:`均等割`, 補足:`調定見込額 402,000×98.9％`

2. **階層構造の復元**
   - CSVの空セルに親の値を継承
   - 「現年課税分」などの同名項目が、異なる親（個人/法人/固定資産税など）の下で正しく区別される

3. **項目名の正規化**
   - スペースの除去: `土 地` → `土地`
   - 表記ゆれを統合

4. **年度別の補足保存**
   - 補足は年度ごとに異なるため、各年度別のカラムに保存

### 出力フォーマット

| カラム | 説明 |
|--------|------|
| 款,項,目,節 | 予算階層（親の値を継承） |
| 説明 | 項目名（補足を除去、正規化済み） |
| R5(確定)〜R8(案) | 各年度の予算額 |
| 補足_R5(確定)〜補足_R8(案) | 各年度の補足情報 |
